# 설계 방향

## 실무형 템플릿 지향

이 프로젝트는 단순한 로그인 앱이 아니라 재사용 가능한 실무형 템플릿을 목표로 합니다. 새로운 프로젝트를 시작할 때 이 구조를 복사하여 인증, 게시판, 파일 업로드, Android WebView 연동의 기본 골격으로 사용할 수 있도록 설계합니다.

## 계층 분리

백엔드는 다음 역할을 기준으로 계층을 분리합니다.

- Controller: HTTP 요청과 응답 처리
- Service: 비즈니스 로직 처리
- Model: 데이터 접근 및 엔티티 표현
- Middleware: 인증, CORS, 요청 검증 등 공통 처리
- Helper: 반복적으로 사용하는 보조 기능
- Core: 라우팅, 요청/응답, 공통 기반 클래스

이 구조는 Laravel이나 Spring Boot 같은 프레임워크의 역할 분리와 유사하므로, 이후 다른 기술 스택으로 전환하더라도 구조를 이해하고 적응하기 쉽습니다.

## API 우선 설계

Android WebView 앱은 백엔드 REST API를 호출하는 클라이언트로 동작합니다. 따라서 화면 구현보다 API의 요청/응답 구조와 인증 흐름을 먼저 안정화합니다.

## 보안 기본값

초기 단계부터 다음 보안 원칙을 반영합니다.

- 비밀번호는 해시로 저장합니다.
- SQL은 Prepared Statement로 실행합니다.
- 사용자 입력값은 검증하고 출력값은 이스케이프합니다.
- 업로드 파일은 확장자, MIME 타입, 크기를 검증합니다.
- JWT Secret, DB 계정 등 민감 정보는 `.env`에서 관리합니다.

## 시연 가능성

면접이나 포트폴리오 시연을 위해 다음 흐름을 명확히 보여줄 수 있도록 구성합니다.

1. Docker로 개발 환경을 실행합니다.
2. Postman으로 회원가입과 로그인을 테스트합니다.
3. JWT 토큰으로 보호된 게시판 API를 호출합니다.
4. 이미지를 업로드하고 프로필 사진을 변경합니다.
5. Android WebView 앱에서 동일한 기능을 확인합니다.
