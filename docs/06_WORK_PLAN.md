# 작업계획서

## 1. 작업 개요

본 문서는 Native PHP Hybrid App 프로젝트의 개발 작업 범위, 단계별 추진 순서, 산출물, 검증 기준을 정리한 실행 계획서입니다.

프로젝트는 Native PHP 8 기반 REST API와 Android WebView 앱을 결합한 하이브리드 애플리케이션을 목표로 합니다. 인증, 게시판, 파일 업로드, 사용자 프로필, Android 연동까지 하나의 흐름으로 구현하여 면접, 포트폴리오, 실무형 샘플 프로젝트에서 설명 가능한 구조를 갖추는 것을 목표로 합니다.

## 2. 작업 목표

- Native PHP 기반 REST API 서버 구조를 설계하고 구현합니다.
- MariaDB와 PDO를 사용하여 안전한 데이터 접근 계층을 구성합니다.
- JWT 기반 회원가입, 로그인, 로그아웃, 인증 검증 흐름을 구현합니다.
- 게시판 CRUD와 이미지 업로드 기능을 구현합니다.
- Android WebView에서 PHP API를 호출하고 사용자 흐름을 확인할 수 있도록 연동합니다.
- Docker, Postman, 문서화를 통해 설치, 실행, 검증 과정을 재현 가능하게 만듭니다.

## 7. 검증 계획

### 7.1 API 검증

- Postman으로 인증, 게시판, 업로드 API를 검증합니다.
- 정상 요청과 오류 요청을 모두 확인합니다.
- HTTP Status Code, 응답 메시지, 응답 데이터 구조를 확인합니다.

### 7.2 보안 검증

- 비밀번호 평문 저장 여부를 확인합니다.
- SQL Injection 방어를 위해 Prepared Statement 사용 여부를 확인합니다.
- XSS 방어를 위해 출력 데이터 처리 기준을 확인합니다.
- 업로드 파일 확장자, MIME Type, 크기 제한을 확인합니다.
- 인증이 필요한 API가 토큰 없이 호출되지 않는지 확인합니다.

### 7.3 Android 검증

- WebView 화면 로딩을 확인합니다.
- 로그인 상태 유지 여부를 확인합니다.
- 게시판 주요 기능을 Android 앱에서 확인합니다.
- 파일 업로드가 Android 환경에서 정상 동작하는지 확인합니다.
- 네트워크 오류 상황에서 사용자에게 적절한 화면이 표시되는지 확인합니다.

## 9. 리스크 및 대응 방안

| 리스크 | 영향 | 대응 방안 |
| --- | --- | --- |
| Android WebView 파일 업로드 처리 지연 | 앱 연동 일정 지연 | 백엔드 업로드 API를 먼저 완성하고 Android는 별도 단계로 검증합니다. |
| 서버 Rewrite 설정 오류 | API 라우팅 실패 | Apache 설정과 `.htaccess` 예제를 문서화합니다. |

| 업로드 보안 취약점 | 보안 리스크 증가 | 확장자, MIME Type, 용량, 저장 경로를 모두 제한합니다. |
