# 테스트 체크리스트

## 1. 개요

본 문서는 Native PHP Hybrid App의 주요 기능을 수동으로 검증하기 위한 체크리스트입니다.

초기 v1.0에서는 Postman을 통한 API 검증과 Android WebView 동작 확인을 중심으로 테스트합니다.

## 2. 테스트 전 준비

- [ ] PHP 서버가 실행 중입니다.
- [ ] MariaDB가 실행 중입니다.
- [ ] `.env` 설정이 완료되었습니다.
- [ ] 데이터베이스 스키마가 적용되었습니다.
- [ ] Composer 의존성이 설치되었습니다.
- [ ] Postman Environment에 `base_url`이 설정되었습니다.
- [ ] Android 테스트 기기 또는 Emulator가 준비되었습니다.

## 3. 공통 API 테스트

### 3.1 Health Check

- [ ] `GET /health` 요청이 200을 반환합니다.
- [ ] 응답의 `success` 값이 `true`입니다.
- [ ] 서버 상태 메시지가 반환됩니다.

### 3.2 공통 응답 형식

- [ ] 성공 응답이 `success`, `message`, `data` 구조를 사용합니다.
- [ ] 실패 응답이 `success`, `message`, `errors` 구조를 사용합니다.
- [ ] 잘못된 경로 요청이 404를 반환합니다.
- [ ] 서버 오류 상황에서 민감한 정보가 노출되지 않습니다.

## 4. 회원가입 테스트

- [ ] 정상 이메일, 비밀번호, 이름으로 회원가입이 성공합니다.
- [ ] 중복 이메일로 회원가입 시 409 또는 422 오류가 반환됩니다.
- [ ] 이메일 누락 시 오류가 반환됩니다.
- [ ] 잘못된 이메일 형식 입력 시 오류가 반환됩니다.
- [ ] 비밀번호 누락 시 오류가 반환됩니다.
- [ ] 8자 미만 비밀번호 입력 시 오류가 반환됩니다.
- [ ] 이름 누락 시 오류가 반환됩니다.
- [ ] 비밀번호가 DB에 평문으로 저장되지 않습니다.

## 5. 로그인 테스트

- [ ] 올바른 이메일과 비밀번호로 로그인이 성공합니다.
- [ ] 로그인 성공 시 `access_token`이 반환됩니다.
- [ ] 로그인 성공 시 사용자 정보가 반환됩니다.
- [ ] 잘못된 비밀번호 입력 시 401 오류가 반환됩니다.
- [ ] 존재하지 않는 이메일 입력 시 401 오류가 반환됩니다.
- [ ] 이메일 누락 시 오류가 반환됩니다.
- [ ] 비밀번호 누락 시 오류가 반환됩니다.
- [ ] 반환된 토큰을 Postman Environment의 `access_token`에 저장합니다.

## 6. 인증 테스트

- [ ] `GET /auth/me`가 정상 토큰으로 성공합니다.
- [ ] 토큰 없이 인증 API 호출 시 401 오류가 반환됩니다.
- [ ] 잘못된 토큰으로 인증 API 호출 시 401 오류가 반환됩니다.
- [ ] 만료된 토큰으로 인증 API 호출 시 401 오류가 반환됩니다.
- [ ] 로그아웃 API가 정상 응답합니다.
- [ ] 클라이언트에서 로그아웃 후 토큰이 제거됩니다.

## 7. 게시판 목록 테스트

- [ ] `GET /boards` 요청이 200을 반환합니다.
- [ ] 게시글 목록이 배열로 반환됩니다.
- [ ] 페이지 정보가 반환됩니다.
- [ ] `page` 파라미터가 동작합니다.
- [ ] `limit` 파라미터가 동작합니다.
- [ ] 게시글이 최신순으로 정렬됩니다.

## 8. 게시글 등록 테스트

- [ ] 정상 토큰으로 게시글 등록이 성공합니다.
- [ ] 제목과 본문이 DB에 저장됩니다.
- [ ] 등록 응답에 게시글 ID가 반환됩니다.
- [ ] 토큰 없이 등록 요청 시 401 오류가 반환됩니다.
- [ ] 제목 누락 시 오류가 반환됩니다.
- [ ] 본문 누락 시 오류가 반환됩니다.
- [ ] 너무 긴 제목 입력 시 오류가 반환됩니다.

## 9. 게시글 상세 테스트

- [ ] 존재하는 게시글 ID로 상세 조회가 성공합니다.
- [ ] 제목, 본문, 작성자 정보가 반환됩니다.
- [ ] 존재하지 않는 게시글 ID로 요청 시 404 오류가 반환됩니다.
- [ ] 삭제된 게시글은 조회되지 않습니다.

## 10. 게시글 수정 테스트

- [ ] 작성자가 게시글을 수정할 수 있습니다.
- [ ] 수정 후 제목과 본문이 변경됩니다.
- [ ] 작성자가 아닌 사용자는 수정할 수 없습니다.
- [ ] 토큰 없이 수정 요청 시 401 오류가 반환됩니다.
- [ ] 존재하지 않는 게시글 수정 시 404 오류가 반환됩니다.
- [ ] 제목 또는 본문 누락 시 오류가 반환됩니다.

## 11. 게시글 삭제 테스트

- [ ] 작성자가 게시글을 삭제할 수 있습니다.
- [ ] 삭제 후 목록에서 제외됩니다.
- [ ] 삭제 후 상세 조회 시 404 오류가 반환됩니다.
- [ ] 작성자가 아닌 사용자는 삭제할 수 없습니다.
- [ ] 토큰 없이 삭제 요청 시 401 오류가 반환됩니다.
- [ ] 존재하지 않는 게시글 삭제 시 404 오류가 반환됩니다.

## 12. 이미지 업로드 테스트

- [ ] 정상 이미지 파일 업로드가 성공합니다.
- [ ] 업로드 응답에 파일 URL이 반환됩니다.
- [ ] 업로드 파일이 서버 디렉터리에 저장됩니다.
- [ ] 파일 메타데이터가 DB에 저장됩니다.
- [ ] 허용되지 않은 확장자 업로드 시 오류가 반환됩니다.
- [ ] 허용되지 않은 MIME Type 업로드 시 오류가 반환됩니다.
- [ ] 제한 용량 초과 파일 업로드 시 오류가 반환됩니다.
- [ ] 토큰 없이 업로드 요청 시 401 오류가 반환됩니다.

## 13. 프로필 사진 테스트

- [ ] 정상 이미지로 프로필 사진 변경이 성공합니다.
- [ ] 내 정보 조회 시 변경된 프로필 이미지 URL이 반환됩니다.
- [ ] 새 이미지 업로드 시 기존 이미지 처리 정책이 정상 동작합니다.
- [ ] 잘못된 파일 형식 업로드 시 오류가 반환됩니다.

## 14. 보안 테스트

- [ ] SQL Injection 형태의 입력값이 쿼리를 변조하지 않습니다.
- [ ] 스크립트 태그 입력값이 화면 출력 시 실행되지 않습니다.
- [ ] `.env` 파일이 웹에서 직접 접근되지 않습니다.
- [ ] 업로드 디렉터리에서 PHP 파일이 실행되지 않습니다.
- [ ] 오류 응답에 DB 비밀번호, JWT Secret 등 민감 정보가 노출되지 않습니다.

## 15. Android WebView 테스트

- [ ] Android 앱이 정상 실행됩니다.
- [ ] WebView가 지정된 URL을 로드합니다.
- [ ] 로그인 화면 또는 시작 화면이 표시됩니다.
- [ ] 로그인 성공 후 인증 상태가 유지됩니다.
- [ ] 게시글 목록을 확인할 수 있습니다.
- [ ] 게시글 상세 화면을 확인할 수 있습니다.
- [ ] 게시글 등록 흐름이 동작합니다.
- [ ] 이미지 업로드가 Android 환경에서 동작합니다.
- [ ] Pull To Refresh가 동작합니다.
- [ ] 네트워크 오류 시 오류 화면 또는 안내가 표시됩니다.

## 16. 완료 체크

- [ ] 회원가입부터 로그인까지 전체 인증 흐름을 검증했습니다.
- [ ] 게시판 CRUD 전체 흐름을 검증했습니다.
- [ ] 파일 업로드와 프로필 사진 변경을 검증했습니다.
- [ ] Android WebView에서 핵심 기능을 검증했습니다.
- [ ] Postman Collection이 최신 API 명세와 일치합니다.
- [ ] README와 docs 문서가 실제 구현과 일치합니다.
